\section{Comparative Analysis of Object Detection using YOLOv5 and YOLOv7}

\subsection{Experimental Setup}

A comparative study using \gls{yolo}v5 and \gls{yolo}v7 to detect and classify hand-drawn electronic circuit components was conducted to determine which model is better-suited for the needs of this project. The dataset consisted of 61 classes, including resistors, capacitors (polarized and unpolarized), diodes, transistors (\gls{bjt} and \gls{fet}), integrated circuits, switches, and more. To ensure a fair comparison, both models were trained and validated on identical data splits.

\section{Extended Object Detection: 15-Class Taxonomy and Multi-Architecture Benchmark}

A key finding from the initial 61-class experiments was that many fine-grained component subclasses (e.g., \textit{capacitor.polarized}, \textit{capacitor.unpolarized}) shared near-identical visual features, causing inter-class confusion. To address this, the 61-class taxonomy was consolidated into a 15-class merged taxonomy by grouping visually similar subclasses. This section presents a comprehensive benchmark of four modern detection architectures on the new 15-class dataset, followed by a holistic comparison against the original 61-class results.

\subsection{Class Taxonomy Consolidation}

The original \gls{cghd} dataset contained 61 fine-grained component classes. Many of these were subvariants of the same physical component and were visually indistinguishable in hand-drawn form. A merge mapping was devised to consolidate these into 15 classes: \textit{capacitor}, \textit{crossover}, \textit{diode}, \textit{gnd} (ground), \textit{inductor}, \textit{integrated\_circuit}, \textit{junction}, \textit{operational\_amplifier}, \textit{resistor}, \textit{switch}, \textit{terminal}, \textit{text}, \textit{transistor}, \textit{voltage}, and \textit{vss}. This consolidation eliminates ambiguity while preserving all electrically distinct component types needed for netlist generation.

\subsection{Experimental Setup (15-Class)}

Four \gls{yolo} architectures were trained and evaluated on the 15-class dataset under uniform conditions:

\begin{itemize}
    \item \textbf{YOLOv7 (retrained):} The same YOLOv7 architecture used in the 61-class experiments, retrained on the merged 15-class taxonomy for 100 epochs with a batch size of 16 and image size of $640 \times 640$ pixels.
    \item \textbf{YOLOv8:} Ultralytics YOLOv8 architecture trained for 100 epochs with identical data splits.
    \item \textbf{YOLOv11:} Ultralytics YOLOv11 architecture trained for 100 epochs.
    \item \textbf{YOLOv26:} A newer Ultralytics variant trained for 100 epochs with patience-based early stopping (patience = 20).
\end{itemize}

All models used identical training/validation/test splits, a confidence threshold of 0.25, and an \gls{iou} threshold of 0.45 during evaluation.

\subsection{Key Observations}

The comprehensive benchmark yields the following insights:

\begin{enumerate}
    \item \textbf{Class taxonomy matters more than architecture.} Consolidating 61 classes to 15 improved YOLOv7's \gls{map}@0.5 from 0.83 to 0.9562 (+15.2\%), whereas switching architectures within the same taxonomy (e.g., YOLOv8 vs. YOLOv26 on 15 classes) produced marginal differences ($<$2\%).
    \item \textbf{YOLOv7 (retrained) is the optimal choice.} It achieves the highest precision (0.9747), recall (0.9350), \gls{map}@0.5 (0.9562), and \gls{map}@0.5:0.95 (0.7266) among all configurations.
    \item \textbf{YOLOv11 is a strong alternative.} With \gls{map}@0.5 of 0.9531 (only 0.3\% below YOLOv7), YOLOv11 offers competitive performance and may be preferred in scenarios requiring its architectural benefits.
    \item \textbf{YOLOv8 and YOLOv26 underperform.} Both lag behind by $\sim$10\% in \gls{map}@0.5, likely due to differences in feature aggregation strategies and training optimization for this specific domain.
    \item \textbf{Convergence speed.} YOLOv7 and YOLOv11 reach 95\% of their peak \gls{map}@0.5 within 40--50 epochs, while YOLOv8 and YOLOv26 require the full 100 epochs and still plateau at lower values.
\end{enumerate}

\section{Comparative Analysis of \gls{ocr} Models}

Two distinct \gls{ocr} architectures were evaluated for extracting component values (e.g., ``10k$\Omega$'', ``100$\mu$F'') from detected text regions: the custom \gls{crnn}-based model trained from scratch, and a fine-tuned TrOCR-small model based on the Vision Transformer architecture.

\subsection{Architecture Comparison}

The custom \gls{crnn} model follows a three-stage pipeline: a \gls{cnn} encoder extracts visual features from the text crop, a bidirectional \gls{lstm} models sequential dependencies, and a \gls{ctc} decoder produces the final character sequence. This model was trained from scratch for 50 epochs using the DTRB framework.

TrOCR-small employs a fundamentally different approach: a Vision Transformer (ViT) encoder processes the input image as a sequence of patches, and a Transformer decoder auto-regressively generates the output text. The pre-trained model (Microsoft's TrOCR-small-printed) was fine-tuned on the circuit text dataset for 3 epochs only, leveraging transfer learning from large-scale printed text recognition.

\subsection{Analysis}

The fine-tuned TrOCR model outperforms the custom \gls{crnn} across all metrics despite requiring only 3 training epochs compared to 50. Key factors contributing to TrOCR's superiority include:

\begin{enumerate}
    \item \textbf{Transfer learning:} TrOCR leverages pre-trained weights from large-scale text recognition datasets, providing strong initial feature representations that generalize well to circuit component text.
    \item \textbf{Attention mechanism:} The Transformer decoder's self-attention captures long-range dependencies more effectively than the \gls{lstm}-based sequential decoding in \gls{crnn}, particularly for multi-character component values.
    \item \textbf{Batch inference:} TrOCR processes all text crops from a single image in one forward pass using GPU parallelism with float16 precision, whereas the \gls{crnn} model processes crops sequentially. This results in significantly faster inference in practice.
    \item \textbf{Robustness:} The \gls{cer} reduction from 0.25 to 0.12 indicates that TrOCR makes half as many character-level errors, directly improving the accuracy of extracted component values for netlist generation.
\end{enumerate}

It is noted that the custom \gls{crnn}'s lower accuracy (65.96\%) was partly attributed to data corruption during preprocessing (as discussed in \cref{fig:ocr-performance-metrics}), where some training crops were entirely black. The TrOCR model, being fine-tuned on cleaned data, did not suffer from this issue. Regardless, the architectural advantages of the Transformer-based approach make TrOCR the superior choice for this pipeline, and it is adopted as the final \gls{ocr} model in the production system.

\section{Junction-Based Wire Detection}

This section presents the experimental results obtained from the proposed junction-based wire detection pipeline applied to circuit diagram images. Each processing stage is evaluated independently and supported with quantitative measurements and visual evidence.

\subsection{Input Image and Preprocessing}

The experiment begins with a high-resolution scanned circuit diagram used as the input image. The original image dimensions are reported in terms of width and height in pixels to establish spatial scale consistency throughout the pipeline.

To enhance structural features and suppress background noise, the input image is converted to grayscale. Otsu's global thresholding method is then applied to automatically compute an optimal threshold value that separates foreground circuit elements from the background. The resulting binary image preserves wires, junctions, and components while eliminating illumination variations.

\subsection{Skeletonization}

Morphological skeletonization is applied to the binary image to reduce all wire segments to single pixel-wide centerlines. This transformation significantly reduces the number of foreground pixels while preserving the topological connectivity of the circuit network.

The reduction in foreground pixel count before and after skeletonization is recorded to quantify structural simplification. Despite aggressive thinning, wire continuity and junction connectivity remain intact, making the skeleton suitable for graph-based analysis.

Object detection is performed using a trained \gls{yolo} model to identify and localize elements in the circuit diagram. Detected objects are classified as follows: Class 0 corresponds to text annotations, Class 1 corresponds to junctions, and all remaining classes correspond to circuit components.

The total number of detected objects per class is reported. To isolate only the wire network, bounding boxes associated with non-junction classes are removed from the skeleton image. This step eliminates component shapes while preserving junction points and interconnecting wires.

Each junction pair is classified based on geometric alignment.
