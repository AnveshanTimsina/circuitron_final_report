\section{Comparative Analysis of Object Detection using YOLOv5 and YOLOv7}

\subsection{Experimental Setup}

A comparative study using \gls{yolo}v5 and \gls{yolo}v7 to detect and classify hand-drawn electronic circuit components was conducted to determine which model is better-suited for the needs of this project. The dataset consisted of 61 classes, including resistors, capacitors (polarized and unpolarized), diodes, transistors (\gls{bjt} and \gls{fet}), integrated circuits, switches, and more. To ensure a fair comparison, both models were trained and validated on identical data splits.

\section{Class Taxonomy Consolidation}

A key finding from the initial 61-class experiments was that many fine-grained component subclasses (e.g., capacitor.polarized, capacitor.unpolarized) shared near-identical visual features, causing inter-class confusion. To address this, the 61-class taxonomy was consolidated into a 15-class merged taxonomy by grouping visually similar subclasses. This section presents findings from training and evaluating four \gls{yolo} architectures (YOLOv7, YOLOv8, YOLOv11, YOLOv26) on the consolidated 15-class dataset along with comparative analysis with the 61-class results.

It is to be noted that, for fairness, all four \gls{yolo} architectures were trained and evaluated on the 15-class dataset under uniform conditions. All models used identical training/validation/test splits, a confidence threshold of 0.25, and an \gls{iou} threshold of 0.45 during evaluation.

The \gls{cer} reduction from 0.25 to 0.12 indicates that TrOCR makes half as many character-level errors, directly improving the accuracy of extracted component values. It is noted that the custom \gls{crnn}'s lower accuracy (66.77\%) was partly attributed to data corruption during preprocessing. Regardless, the architectural advantages of the Transformer-based approach make TrOCR the superior choice for this pipeline. However, the inference speed of TrOCR has found to be considerably slower than the custom OCR model. This implies there is inherently a trade-off between accuracy and inference speed when choosing between the custom OCR and TrOCR models for this project. Keeping this in mind, both models are retained in the codebase and can be switched based on user preference.

\section{Junction-Based Wire Detection}

\begin{figure}[H]
    \centering
    \includegraphics[width=\textwidth]{src/images/figures/otsu_thresholding.png}
    \caption{Otsu's Global Thresholding}
    \label{fig:otsu-thresholding}
\end{figure}

This section presents the experimental results obtained from the junction-based wire detection pipeline applied to circuit diagram images. A high-resolution scanned circuit diagram is used as the input image. The original image dimensions are reported in terms of width and height in pixels to establish spatial scale consistency throughout the pipeline. To enhance structural features and suppress background noise, the input image is converted to grayscale. Otsu's global thresholding method is then applied to automatically compute an optimal threshold value that separates foreground circuit elements from the background. The resulting binary image preserves wires, junctions, and components while eliminating illumination variations as shown in \cref{fig:otsu-thresholding}.

Morphological skeletonization is performed on the binary image to reduce all lines to a single-pixel width while maintaining connectivity. This process simplifies the subsequent analysis of wire paths and junctions. The skeletonized image is then overlaid with bounding boxes obtained from the object detection stage (i.e, YOLOv7) to identify potential endpoints where wires intersect component boundaries. Detected endpoints are marked on the skeleton, and their spatial coordinates are recorded for further processing.

\begin{figure}[H]
    \centering
    \includegraphics[width=\textwidth]{src/images/figures/bounding_box_overlay.png}
    \caption{Bounding Box Overlay}
    \label{fig:bbox-overlay}
\end{figure}

\cref{fig:bbox-overlay} shows the skeletonized image with bounding boxes overlaid. Intersection points between wires and component bounding box edges are computed to identify electrical connectivity. The total number of intersection points is reported along with a directional breakdown for horizontal and vertical wires intersecting specific component edges. These intersections form the basis for reconstructing circuit connectivity graphs and CircuitJS-compatible text files.

However, this junction-based method has a very important shortcoming: it fails to handle diagonal lines. This is because the bounding box intersection method relies on the assumption that wires will intersect component boundaries along horizontal or vertical axes. Diagonal wires may not intersect the bounding box edges in a predictable manner, leading to missed connections and incomplete graph construction. Therefore, this method is rejected in favor of a more robust path-finding approach that can handle arbitrary wire orientations and junction configurations.

The frontend of the Circuitron web application provides an interactive environment for circuit design, simulation, and analysis. It enables users to construct electrical circuits using a drag-and-drop schematic editor with grid-based alignment, snap-to-grid functionality, and real-time wire routing. Components such as passive elements, active devices, and sources can be configured through a properties panel that allows precise control over electrical parameters and placement. 

The system integrates simulation controls for transient analysis, allowing users to define time parameters, execute simulations, and visualize results through dynamically rendered voltage-time plots. Measurement probes support real-time signal monitoring with automated computation of key metrics, including maximum, minimum, peak-to-peak range, and \gls{rms} values. Additionally, the frontend supports exporting simulation data and visualizations in standard formats, enabling seamless documentation and further offline analysis.
